# Pilot Workspace Operations

These are the missing pieces that make a generated pilot workspace usable day-to-day.

## Minimum operational loop
For each active pilot workspace:
1. review new inbound examples or forwarded leads
2. classify urgent vs low-priority
3. draft suggested replies
4. flag stale follow-up opportunities
5. log anything that was wrong or noisy
6. update the client workspace files if durable guidance changed

## Key files to keep current
- `SAMPLES.md`
- `TONE.md`
- `BOUNDARIES.md`
- `WORKFLOWS.md`
- `LOG.md`
- `DAILY-CHECKLIST.md`
- `LEAD-QUEUE.md`
- `DRAFTS.md`

## Rule
The pilot workspace should not just be a folder of setup docs. It should become the operating surface for the pilot.
